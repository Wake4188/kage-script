# Contributing to Kage

Thanks for helping improve Kage Script.

## Development

1. Install dependencies with `bun install` or `npm install`.
2. Start the dev server with `npm run dev`.
3. Run tests with `npm test` before opening a pull request.

## Guidelines

- Keep changes focused on educational value, SEO quality, and performance.
- Avoid duplicating existing routes or content.
- Preserve the minimal visual style of the site.
- Add or update tests for logic changes.

## Pull requests

Please include a short summary, the motivation behind the change, and any relevant screenshots or links.
